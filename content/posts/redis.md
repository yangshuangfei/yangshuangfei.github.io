## 1. 通用键 (Key) 命令

这些命令不限数据类型，是对“名字”本身的操作。

| **命令**      | **示例**          | **注释**                                    |
| ------------- | ----------------- | ------------------------------------------- |
| **`KEYS \*`** | `KEYS user:*`     | 查找所有符合模式的键 (生产环境慎用，会阻塞) |
| **`EXISTS`**  | `EXISTS mykey`    | 检查键是否存在 (1存在, 0不存在)             |
| **`DEL`**     | `DEL mykey`       | 删除一个或多个键                            |
| **`EXPIRE`**  | `EXPIRE mykey 60` | 设置过期时间 (单位：秒)                     |
| **`TTL`**     | `TTL mykey`       | 查看剩余存活时间 (-1永久, -2已过期)         |
| **`TYPE`**    | `TYPE mykey`      | 返回该键存储的数据类型                      |

## 2. 字符串 (String) 命令

最常用的类型，用于存文本、数字、JSON串或二进制

| **命令**    | **示例**              | **注释**                                |
| ----------- | --------------------- | --------------------------------------- |
| **`SET`**   | `SET name "Gemini"`   | 设置键值对                              |
| **`GET`**   | `GET name`            | 获取键的值                              |
| **`SETEX`** | `SETEX code 60 "123"` | 设置值并指定过期时间 (原子操作)         |
| **`SETNX`** | `SETNX lock "ok"`     | **键不存在才设置** (常用作分布式锁基础) |
| **`INCR`**  | `INCR view_count`     | 将数字值自增 1 (原子计数)               |
| **`MGET`**  | `MGET k1 k2 k3`       | 批量获取多个键的值                      |

## 3. 哈希 (Hash) 命令

适合存储对象（如用户信息、配置信息）。

| **命令**      | **示例**                   | **注释**                       |
| ------------- | -------------------------- | ------------------------------ |
| **`HSET`**    | `HSET user:1 name "Alice"` | 设置哈希字段的值               |
| **`HGET`**    | `HGET user:1 name`         | 获取哈希字段的值               |
| **`HGETALL`** | `HGETALL user:1`           | 获取该哈希中**所有的**字段和值 |
| **`HDEL`**    | `HDEL user:1 age`          | 删除哈希中的某个字段           |
| **`HEXISTS`** | `HEXISTS user:1 name`      | 判断字段是否存在               |



## 4. 列表 (List) 命令

双向链表，常用于队列、栈或最新消息列表。

| **命令**     | **示例**             | **注释**                         |
| ------------ | -------------------- | -------------------------------- |
| **`LPUSH`**  | `LPUSH task_q "t1"`  | 从左侧（头部）插入元素           |
| **`RPUSH`**  | `RPUSH task_q "t2"`  | 从右侧（尾部）插入元素           |
| **`LPOP`**   | `LPOP task_q`        | 移除并返回左侧第一个元素         |
| **`LRANGE`** | `LRANGE task_q 0 -1` | 获取指定范围元素 (0 -1 表示全部) |
| **`LLEN`**   | `LLEN task_q`        | 获取列表长度                     |



## 5. 集合 (Set) 命令

无序、唯一的字符串集合。自带去重。

| **命令**        | **示例**               | **注释**                          |
| --------------- | ---------------------- | --------------------------------- |
| **`SADD`**      | `SADD tags "java"`     | 向集合添加元素                    |
| **`SMEMBERS`**  | `SMEMBERS tags`        | 返回集合所有元素                  |
| **`SISMEMBER`** | `SISMEMBER tags "c++"` | 判断元素是否在集合中 (1是, 0否)   |
| **`SREM`**      | `SREM tags "java"`     | 移除集合中的元素                  |
| **`SINTER`**    | `SINTER s1 s2`         | 求两个集合的**交集** (找共同好友) |



## 6. 有序集合 (ZSet) 命令

每个元素关联一个 **Score（分数）**，自动按分数排序。

| **命令**        | **示例**                | **注释**               |
| --------------- | ----------------------- | ---------------------- |
| **`ZADD`**      | `ZADD rank 100 "user1"` | 添加元素并指定分数     |
| **`ZRANGE`**    | `ZRANGE rank 0 -1`      | 按分数从小到大返回元素 |
| **`ZREVRANGE`** | `ZREVRANGE rank 0 9`    | 返回分数最高的前 10 名 |
| **`ZSCORE`**    | `ZSCORE rank "user1"`   | 获取某个元素的分数     |
| **`ZREM`**      | `ZREM rank "user1"`     | 移除元素               |

## 7. 系统与管理命令

用于查看 Redis 状态或清空数据。

- **`PING`**: 返回 `PONG`，测试连接是否正常。
- **`INFO`**: 查看 Redis 的运行状态、内存使用、版本信息等。
- **`FLUSHDB`**: **清空当前数据库**所有数据（慎用！）。
- **`DBSIZE`**: 返回当前数据库的 key 总数。





# Redis 底层数据结构

### 1. 总览：Redis 对象 (redisObject)

在 Redis 中，当你执行 `SET key value` 时，Redis 实际上创建了两个对象：一个是键对象，一个是值对象。它们都由 `redisObject` 结构体表示。它像是一个“外壳”，包裹着具体的底层数据。

```C
typedef struct redisObject {
    unsigned type:4;        // 类型 (String, List, Hash, Set, Zset)
    unsigned encoding:4;    // 编码 (底层使用的是哪种数据结构)
    unsigned lru:24;        // LRU/LFU 计时 (用于内存淘汰)
    int refcount;           // 引用计数
    void *ptr;              // 指向底层数据结构的指针
} robj;
```

- **Type**: 也就是我们在 Redis 层面看到的 `string`, `list`, `hash` 等。
- **Encoding**: **这才是今天的重点**。它决定了实际上数据是怎么存的（比如是存成链表，还是跳表，还是压缩列表）。



## 2. SDS (Simple Dynamic String) - 简单动态字符串

**对应类型：** `String`

Redis 没有直接使用 C 语言的字符串（以 `\0` 结尾的字符数组），而是自己构建了 SDS。

```C
struct sdshdr {
    int len;     // 已使用的字节长度 (获取长度 O(1))
    int alloc;   // 已分配的总长度
    char buf[];  // 实际存储内容的字符数组
};
```

### 核心优势（注释）：

- **O(1) 获取长度**：C 字符串需要遍历整个数组，SDS 直接读 `len`。
- **防止缓冲区溢出**：修改前会检查 `alloc`，空间不足自动扩容。
- **减少内存分配次数**：通过“空间预分配”和“惰性空间释放”，避免频繁向操作系统申请内存。
- **二进制安全**：可以存储图片、压缩文件等二进制数据，因为它不以 `\0` 判断结束，而是看 `len`。

------

## 3. ZipList (压缩列表) 与 Listpack (紧凑列表)

**对应类型：** `List`, `Hash`, `ZSet` (数据量较小时使用)

### 设计初衷：

为了**极致节省内存**。它是一块**连续的内存地址**，像数组一样排列，但每个节点的大小可以不同。

### 核心演进（注释）：

- **ZipList (老版本)**：每个节点记录了前一个节点的长度。
  - *痛点*：如果前面的节点长度变了，可能导致后面所有节点跟着重排（**连锁更新**），性能抖动。
- **Listpack (Redis 7.0+ 完全替代)**：去掉了对前一个节点长度的依赖。
  - *优势*：彻底解决了连锁更新问题，结构更简单，更安全。

------

## 4. Dict / HashTable (字典/哈希表)

**对应类型：** `Hash`, `Set` (数据量大时)

### 结构特点：

Redis 的字典采用拉链法解决哈希冲突，并有两个全局哈希表（`ht[0]` 和 `ht[1]`）。

### 渐进式 Rehash（核心注释）：

- **为什么要两个表？** 当哈希表需要扩容时，Redis 不会一次性搬迁，因为数据量太大（如千万级）会卡死。
- **怎么搬？** 每次对该字典进行增删改查时，顺便搬一小部分数据从 `ht[0]` 到 `ht[1]`。
- **结果**：将巨大的计算量平摊到每一次操作中，保证了响应的平滑性。

------

## 5. SkipList (跳跃表)

**对应类型：** `ZSet` (数据量大时)

### 原理简述：

跳表是在链表的基础上，增加了**多级索引**。

### 核心优势（注释）：

- **查询效率**：平均时间复杂度为 O(\log N)，性能媲美红黑树。
- **实现简单**：比红黑树更容易实现和调试。
- **范围查询极快**：因为底层是个有序的双向链表，找到起点后直接往后拉就行。

------

## 6. QuickList (快速列表)

**对应类型：** `List`

### 结构：

QuickList 是 **ZipList** 和 **双向链表** 的结合体。

### 巧妙之处（注释）：

- 链表的每个节点都是一个 ZipList（或 Listpack）。
- **平衡点**：普通链表附加指针多，浪费内存；ZipList 太大，分配内存慢。QuickList 取了折中，既保证了插入删除的效率，又节省了空间。

------

## 7. IntSet (整数集合)

**对应类型：** `Set` (仅包含整数且数量不多时)

### 核心注释：

- **内存紧凑**：如果集合里全是整数，Redis 会用一块连续的数组存，非常省空间。
- **自动升级**：如果你存的是 16 位整数，突然塞进一个 32 位的，它会自动把整个数组升级到 32 位，但**不支持降级**。

------

## 总结：数据类型与底层结构的对应关系

| **用户视角的数据类型** | **数据量较少/元素较小时 (追求内存)**    | **数据量较大/元素较大时 (追求速度)** |
| ---------------------- | --------------------------------------- | ------------------------------------ |
| **String**             | **Int** (如果是纯数字)                  | **SDS** (Raw / Embstr)               |
| **List**               | **Quicklist** (Listpack 组成的双向链表) | **Quicklist**                        |
| **Hash**               | **Listpack** (旧版为 Ziplist)           | **Dict** (Hashtable)                 |
| **Set**                | **Intset** (整数集合)                   | **Dict** (Value 为 null)             |
| **ZSet**               | **Listpack** (旧版为 Ziplist)           | **Skiplist + Dict**                  |
